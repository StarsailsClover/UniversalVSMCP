## 🎯 关键决策点

### 决策 1: v27.0 是否保持 .NET 独占？

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **.NET 独占** | 性能好，一致性强 | 跨平台受限 | ✅ 保持 |
| **多运行时** | 更灵活 | 维护成本高 | ❌ v28 再考虑 |

**决策**: v27 保持 .NET 8 独占，v28 评估多运行时

### 决策 2: VS Code 原生 MCP vs 扩展？

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **原生 MCP** | 零扩展安装 | 功能受限 | 🔄 双轨支持 |
| **扩展** | 功能完整 | 需安装 | 🔄 双轨支持 |

**决策**: 双轨支持，优先原生 MCP，回退扩展

### 决策 3: 开源 vs 商业？

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **MIT 开源** | 社区贡献 | 商业化难 | ✅ 保持 |
| **商业版** | 收入 | 社区分裂 | 📋 v28 评估 |

**决策**: 保持 MIT，v28 评估商业支持版

## ⚠️ 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| MCP 协议不普及 | 高 | 中 | 同时支持非 MCP 模式 |
| VS API 变更 | 高 | 低 | 抽象层隔离，快速适配 |
| 安全漏洞 | 高 | 低 | 持续审计，快速响应 |
| 资源不足 | 中 | 中 | 开源社区，企业赞助 |
